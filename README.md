# SparseAttnLab

## Overview

SparseAttnLab targets one of the core bottlenecks in large language model inference: the cost of attention. As context lengths continue to grow, standard autoregressive attention becomes increasingly expensive in computation, memory footprint, and latency. Although many sparse-attention ideas are promising at the algorithmic level, they often lack a unified experimental framework, clean engineering interfaces, and a practical path into real model inference. This project is motivated by the need to bring those pieces together in a single, systematic codebase.

Rather than implementing only one sparse pattern or one kernel, the goal of this repository is to provide a complete workflow that spans algorithm prototyping, benchmarking, kernel development, integration with real LLMs, and sparse-architecture search. To support that goal, the repository is organized into three layers: `sparse_attentions/` for pattern and backend abstractions, `sparse_attention_bench/` for layer-level and end-to-end experiments, and `sparse_llm/` for integrating sparse attention into real models and evaluating it under realistic generation and perplexity settings. This structure allows developers to first validate whether a sparse design is directionally sound before investing in deeper kernel optimization or model-level integration.

At the current stage, the framework has already produced several representative engineering and research outcomes. We have implemented a general Triton-based sparse matrix kernel that can accept arbitrary retained `QK^T` patterns, providing a unified compute backend for quickly validating different sparse designs; compared with the baseline implementation, this kernel achieves roughly a 3x improvement in GPU utilization efficiency. On the model side, we have integrated the BigBird pattern into Qwen3-4B and observed stable inference speedups while keeping perplexity degradation within 10%, reaching an average 1.5x speedup and a peak speedup of 1.6x. In experiments on sparse model structure, we also observed that applying sparsity to lower layers tends to preserve perplexity more faithfully, while sparsifying higher layers tends to produce larger speed gains. These results suggest that the project is already useful not only for sparse-attention prototyping, but also for deployment-oriented inference studies on real models.

More importantly, the framework itself has strong research and engineering potential. The repository already includes a fairly complete foundation: unified abstractions for multiple sparse patterns and backends, YAML-driven sweep benchmarking, end-to-end Toy Decoder evaluation, Triton and CUDA kernel entry points, Qwen3 sparse-attention integration, and grouped-query sparse-architecture search. New sparse-attention methods therefore do not need to build experimental infrastructure from scratch; they can be plugged directly into the existing framework and evaluated for speed, memory, accuracy, and integration cost. The framework also supports system-level telemetry and several evaluation modes that are closely aligned with real inference workloads, including WikiText-based perplexity testing, decode-stage throughput measurement, first-token latency analysis, and comparisons across latency, memory, and accuracy. For grouped-query attention in LLMs, the framework can already assign sparsity and pattern settings at the granularity of each KV group, enabling analysis of how different models, layers, and KV groups respond to sparsification. It already supports experimentation on the Qwen3 family and provides a foundation for exploring kernel parameters and sparse structure design across models of different scales.
